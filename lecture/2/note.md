x86 默认为小端存储

例如存储日期 1970-01-02，存储情况为

02 01 70 19

但也不是完全倒着，只是存储的位置有所改变

对于字长的意义，这里解释一下：对于一个计算机来说，字长就是能一次性并行处理的位数，比如对于32位系统，字长就是32，对于64位系统，字长就是64。

但是，在实际的AT&T语法规定中，w(word)就是指16位，l(long)就是指32位，q(quad)就是指64位。是从最基础的16位得来

例如, movl %ebp,%eax就是指把两个长度为32位的寄存器做mov指令

函数调用其实很简单

首先理解寄存器ebp,esp,eip

ebp extented base pointer

esp extented stack pointer

eip extented instruction pointer

eax-edx general register,存各种整数和地址

ebp和esp划定了当前调用函数的申请栈空间，其中ebp指栈底(高地址),esp指栈顶(低地址)，而eip则指向code区域对应的调用函数本体

相当于栈空间是做蛋糕的原材料，而eip指向的code区域则是做蛋糕的说明书(instruction)

基本思路是：记得自己要干什么，所以每调用一个新函数，都要保存之前栈帧的地址和指向的代码的地址。esp不用特地保留，他会随着当前函数栈空间的释放自动往上走，所以需要保留的就是eip和ebp

先存old eip(又称呼为rip),然后move eip指向被调用的函数

然后再保存old ebp,又称呼为sfp(stack frame pointer)，然后move ebp到新的栈帧顶，即旧的栈帧底(指向rip存放的位置)

干完所有事之后move esp,直到碰到保存的ebp,然后return ebp，再return eip,恢复原状

要理解一个函数到底是保存了什么寄存器，返回了什么寄存器其实很简单，只要看eip指向的函数期间干了什么，就能明确任务分配

然后函数的参数调用默认遵循cdecl调用规范，即从右往左，压入栈中(目前只考虑x86 32位的情况)

```
内存高地址 (High Address)
            ^
            |
            | ... (上层函数的栈帧) ...
            |
+-------------------------------------------+
|             参数 b (Arg 2)                |  <-- EBP + 12
|          (值为 2，最先压栈)                 |
+-------------------------------------------+
|             参数 a (Arg 1)                |  <-- EBP + 8
|          (值为 1，最后压栈)                 |
+-------------------------------------------+
|           返回地址 (Return Address)        |  <-- EBP + 4
|       (Call 指令自动压入，func 结束后跳回这里) |
+-------------------------------------------+
|           旧的 EBP (Old EBP)              |  <-- EBP (基址指针指向这里)
|      (保存上一层函数的栈底，方便恢复现场)      |
+-------------------------------------------+
|           局部变量 x (Local 1)            |  <-- EBP - 4
|               (值为 10)                   |
+-------------------------------------------+
|           局部变量 y (Local 2)            |  <-- EBP - 8
|               (值为 20)                   |
+-------------------------------------------+
|          保存的寄存器 (如 EBX, ESI)         |  <-- 编译器优化可能存在
|          (保护现场，非必须，视情况而定)       |
+-------------------------------------------+
|                                           |
|             (栈顶空闲区域)                 |  <-- ESP (栈顶指针指向这里)
|                                           |
            |
            v
      内存低地址 (Low Address)
```

一些基本的汇编函数

push A 往下移esp,然后把A压入栈顶

push ebp就是保存之前旧的栈帧的栈底，所以一般函数的汇编代码开头都是这个

jmp A 将eip的位置设为A

call A 调用函数A(实际上A也可以直接设为地址)，具体来说，call先把下一条指令(**不忘初心**)压入堆栈，即push B，然后直接jmp A。一般情况下，A最后清理完自己的栈帧后会有ret指令，esp会到达A的栈底，而栈底存放的就是B，即返回地址。这个时候会执行ret指令，eip的位置设为B，继续执行原函数的操作


movl A B ;把A的值给B

popl A ;esp指针上移4字节(l代表的就是long,4字节)，并把原先esp内存的值赋给 A

leave 很简单，退出栈帧，也就是 

> movl %ebp %esp 
> popl %ebp

ret: popl %eip

一般函数最后都是

leave

ret

来保证栈帧完整退出并复原状态

常量引用时带美元符号，例如$1,$0x8，寄存器引用时带%，不再赘述

(A)表示把A存放的数据当成地址来看，解引用，类似于\*A,比如(%eip)，前面加数字表示A的地址改变后解引用，比如12(A)表示的意思是\*(A+12)，也就是说，(A)相当于默认偏移量为0

一些术语

SFP saved frame pointer: saved %ebp on the stack

OFP old frame pointer: old %ebp from the previous stack frame

RIP return instruction pointer: return address on the stack

## 总结

从头到尾调用一个函数要干三件事

0. 调用方将参数压入自己的栈中(也就是说函数参数一直保存于栈中)
1. call foo要做两件事，第一件是将rip(也就是call foo之后的一条指令的地址)压入栈中，第二件就是eip移动到foo所在的位置(接下来的操作都是foo函数的一部分)
2. esp下移，用来存下当前的ebp
3. 存下当前的ebp(sfp)
4. 把ebp挪到当前esp的位置，构成新的栈帧的栈底(movl %ebp,%esp)
5. 执行完源代码中函数的所有操作，有返回值的情况下存入eax
6. 清理栈帧(靠esp的移动)
7. esp挪到栈底(与ebp一同指向sfp)
8. leave操作(把esp当前指向的sfp给ebp，并上移，也就是 popl %ebp)
9. ret操作(popl %eip)(foo函数内容结束)
10. eip此时指向的指令：有参数时esp上移，清理函数的参数，没有就执行后面的操作，调用流程结束。
