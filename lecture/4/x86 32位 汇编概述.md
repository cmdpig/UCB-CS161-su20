## Register(寄存器)

eax,ebx,ecx,edx在上古(16位和8位)时代都有其特殊含义，如eax的a是“accumulator”，在算术运算中用作累加器。

不过现在他们的意义已经跟名字没关系了，而且地位也是相等的，我可以用eax来返回函数值，也可以用ebx来返回函数值。

还有为了兼容上古程序保留下来的一些寄存器，例如AX,AH等。通常是借助使用eax的低位来实现的。

## Memory and Addressing Modes

### 声明静态变量
.DATA区域就是用来定义static data(静态数据)的，在C中，通常是全局变量，带有static修饰符的变量和常量

例子：

```asm
.DATA
var DB 64    ;声明一个字节，存放在var位置的变量，且初始值为64
var2 DB ?     ;声明一个字节，存放在var2位置的变量，且未经过初始化
    DB  10  ;声明一个字节，同时没有名字(label)的变量，位置在var2+1处
```

除此之外，还有别的声明大小，DW代表2个字节，DD代表4个字节

声明一个序列时，可以用dup初始化，格式类似于`4 DUP(2)`

```asm
array DD 10 DUP(4)    ;声明10个4字节数据，每个字节赋值为4(十进制)
```

## calling convention(调用约定)

调用约定规定了调用函数(caller)和被调用函数(callee)如何保证前后状态的一致性，一般是c language calling convention，这个应该已经成为事实上的标准了。

调用者(caller)要做的事：
1. 存入当前的寄存器的值（保存当前状态）
2. 从右到左把参数压入栈，这样支持可变参数（一般参数数量由第一个）
