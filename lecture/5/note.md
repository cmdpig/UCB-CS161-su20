## 密码学

>密码学是什么？

以往的密码学定义都是以保证通信安全为基础的，但现在今非昔比

从当下角度看，密码学是一种能够以**正式**的方式来保证信息不被攻击者窃取或篡改

这里，**正式**的意义是我们用数学的方式，从理论上证明他的安全性，而不只是单纯从经验上说明其安全性。毕竟，正常运行一千万次的系统不能保证第一千万零一次不出问题。

密码学主要有三个目标

>[!success] 三个目标
>1. **confidentiality**，机密性。就是防止攻击者读取到我们的私人数据
>2. **integrity**，完整性。我们需要有手段来防止攻击者更改数据。换句话说，攻击者不能在我们不知道的情况下更改数据。
>3. **authenticity**，真实性。即保证接受到的消息为可信方，而不是中间者。

密码学将围绕着三个问题提出各种解决方案。

最经典的两个加密体系：对称加密和非对称加密。

在安全的世界中，仅仅靠隐藏(obscurity)是不够的。必须要假想攻击者知道所有的算法细节和运作原理，但是依然难以攻破。这样才是安全的。

这就是回到了安全原则：不要把安全寄托在隐蔽性上。

有一个类似版本是 Kerkhoff princple(基尔霍夫定律)，也是说明在加密体系中，唯一要保护的只有自己的密钥，算法本身应该被公开。而且，这个算法体系应该要设计的更改密钥是一件比较容易的事。

这也是一个密码体系能被承认有效的必要条件：只有他先经历了大部分密码分析师的检查和改进，这个密码体系才是有效的。在安全中，**可信度**这个要求很重要。

## symmetric encryption(对称加密)

### 模型

简单思路，两个人拿相同的key，一个人对消息M做C=Enc(key,M)，另一个人受到C后M'=Dec(key,C)即可

**假定**在这个加密体系中，我们不能加密过长的密码，因为会失去信息。所以有一个静态明文最大长度$n$，且加密后密文长度与明文长度相同。

然后，为了防止浪费空间(较短的明文输出过长的密文)，所以限制其长度有必要。但目前不考虑具体多长。

现在，拥有一个最大明文长度为$n$的对称加密模型，而这个模型应该由以下三个算法生成

>1. keygen()，负责生成key，理论上来说应该是随机的，这样符合基尔霍夫定律
>2. Enc(key,M)
>3. Dec(key,C)
>后两个的作用是显然的，且我们也能从中看出其对称性(由前文可知)

到此为止，我们还没有谈及其安全性，而这就是接下来要讲的、

###  安全性

朴素的想法：拿到了C,无法解出M。但是年轻人，**too naive!**

假设有一个**确定性加密**的数据库，这里解释一下确定性加密和非确定性加密的概念


| 确定性加密                                        | 非确定性加密                           |
| -------------------------------------------- | -------------------------------- |
| 明文和密钥固定时，每一次加密输出的密文均相同                       | 明文和密钥固定时，每一次加密输出的密文不同            |
| 不引入随机数                                       | 引入随机数，且把随机数放入密文中，方便解密            |
| 不满足“语义安全”，即相同的明文对应不同的密文，可以根据出现频率或得到的部分明文进行攻击 | 满足“语义安全”，攻击者无法根据出现频率或得到的部分明文进行攻击 |




1. 可以解出部分信息，例如一串身份证的加密，我无法解密出完整的身份证，但是可以解出你的属地(开盒😈)，这是属于部分信息(partial info)泄漏
2. 从partial info,可能可以直接解出完整密文，比如ph比赛里的通灵(

综上，我们的加密系统要做到不能有任何partial info泄漏。

但是，怎么严格定义partial info泄漏？要泄漏到什么程度才能说是不安全的。

这里有一个概念叫区分(**distinguishing**)。形式话的说，设两个概率分布$D_{0}$和$D_{1}$，有一个未知来源的样本$x$，区分器(就是指区分)猜测这个样本来源于哪里

若攻击者能够以显著高于$\frac{1}{2}$的概率正确判断样本来源(拥有**区分优势**)，则认为该密码存在区分攻击，不满足安全性。

假设区分器为$A$，区分器是一个算法，输入一个样本$x$，输出一个比特$b\in \{ 0,1 \}$猜测来自哪个分布，区分器的**优势(advantage)** 定义为

$$
\text{Adv}(\mathcal{A}) = \left| \Pr[\mathcal{A}(x)=1 \mid x \leftarrow D_1] - \Pr[\mathcal{A}(x)=1 \mid x \leftarrow D_0] \right|
$$

其中：
- $\Pr[\mathcal{A}(x)=1 \mid x \leftarrow D_1]$ 是当样本来自 $D_1$ 时 $\mathcal{A}$ 输出 1 的概率(即正确识别 $D_1$ 的概率)。
- $\Pr[\mathcal{A}(x)=1 \mid x \leftarrow D_0]$ 是当样本来自 $D_0$ 时 $\mathcal{A}$ 错误地输出 1 的概率(即误报为 $D_1$ 的概率)。

通常，取$D_{0}$为理想随机输出，$D_{1}$为密码系统输出。由此可见，优势越大，越不安全。

密码学要求，对于任何**多项式时间区分器**，优势都是**可忽略的**，即小于某个拥有安全参数的倒数多项式函数

在当下这个场景中，我们说部分信息泄露，就是指可以**区分**信息，比如设$D_{0}$为癌症，$D_{1}$为流感，理想情况下，优势为0，即跟随机选的没区别。但是优势接近1时，我就能区分哪些是癌症哪些是流感了。

## IND-CPA

这样，理解IND-CPA就很简单了。IND-CPA是一种测试语义安全的"game"(真的很像游戏)

IND是 indisguishability，CPA就是chosen plain attack，下文都有体现

具体来说，有一方挑战者(challenger)和一方攻击者(attacker)。

挑战者事先拥有一个加密系统，并保持密钥不变。攻击者可以事先获取到若干对“明文-密文对”（获取对应分布）

随后，攻击者两个发出**长度相同**的明文$M_{0},M_{1}$(依据之前假设，加密不改变长度),挑战者随机选择一个明文$M_{b}$并加密，输出密文$C$，攻击者需要猜测$C$是由什么明文$M_{b'}$加密的

若猜对的概率，即$\Pr[b'=b ]\leq \frac{1}{2}+\varepsilon$，则说明是**语义安全**的，$\varepsilon$一般得小于$\frac{1}{2^{128}}$

一个反直觉的情况是，有一些加密系统可以通过这个game,但是没什么卵用。比如这个加密只会随机翻转一些比特位，那猜对的概率一定严格等于$\frac{1}{2}$，但是我自己也无法解出密文，那就没有任何意义。

>[!warning] 注意
>$M_{0},M_{1}$可以来自之前选择的“明文-密文对”，所以无特殊改变选取密文的方式(如不能与之前相同)，任何确定性的加密系统都不能通过CPA


## OTP
OTP，one time pad，一次性密码本。这是一种理论上绝对安全的加密方法，但正如名字所说，密钥用一次就没用了，必须要换新的，~~所以会拿个小本本记很多可信密钥~~

算法很简单：

1. 随机生成密钥key
2. C=M xor key
3. 传给同样拥有key的人，M=C xor key

这样就好了。只要只用一次，这个算法就是理论与随机噪声无异。

来到CPA，若去除前面的多次测试，防止被拿到key，则我们可以得到两个可能

$$
\begin{aligned}
C=M_{0}\otimes   \underbrace{(M_{0}\otimes K)}_{K_{0}}\\
C=M_{0}\otimes   \underbrace{(M_{1}\otimes K)}_{K_{1}}
\end{aligned}
$$

所以，我们就有对key就有两种猜测，$K_{0}$和$K_{1}$。由于我们保证key是随机生成的，所以$K_{0}$和$K_{1}$是等可能的，我们无法把它与噪声区分开，所以猜对概率是严格的$\frac{1}{2}$。

但是，由于是一次性的特性，所以适用范围很苛刻，一般是国家元首间的红色电话可能会用到。

## NOTE的补充

block cipher，块加密，是一种对称加密算法

(我认为这不算算法，更多的算一种**元算法**，因为没有给出具体的加密算法，需要填充)

理想情况下，把明文划分为固定大小的块(64或者128位)，然后对每一个块进行加密

ECB，electronic code book，最简单的模式，只是对每一个块分别进行加密，也就是说，若$M_{i}=M_{j}$，则$C_{i}=C_{j}$。所以这个可以泄露有关明文的信息，例如出现频率。

CBC，cipher block chaining，就是类似区块链。先设置一个随机值(官方术语叫初始向量，initial vector)$IV$，然后设$C_{0}=IV$，$C_{i}=\text{Enc}(C_{i-1}\otimes M_{i},\text{key})$，这样可以有效防止上述的情况。

OFB，output feedback mode，也是设置一个$IV$，然后设$Z_{0}=IV$，$Z_{i}=\text{Enc}(Z_{i-1},\text{key})$，重复加密出$n$个，然后$C_{i}=M_{i}\otimes Z_{i}$，类似于OTP的思想。但是当attacker得知了其中一个块的明文，那么他就可以伪造相应的密文。CBC也是类似的，所以并不十分安全

CM，counter mode，跟OFB差不多，区别在于$Z_{i}=\text{Enc}(IV+i,key)$，这样可以并行处理，大大提高效率
